94 Binary Tree Inorder Traversal

Medium

  • Binary Tree

Problem

Given a binary tree, return the_inorder_traversal of its nodes’ values.

Example:

Input:
 [1,null,2,3]
   1
    \
     2
    /
   3


Output:
 [1,3,2]

Follow up:Recursive solution is trivial, could you do it iteratively?

Solution

Recursive:

import java.util.ArrayList;
import java.util.List;

/**
 * Definition for a binary tree node.
 * public class TreeNode {
 *     int val;
 *     TreeNode left;
 *     TreeNode right;
 *     TreeNode(int x) { val = x; }
 * }
 */

class BinaryTreeInorderTraversal_Recursive {
    public class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;
        TreeNode (int x) { val = x; }
    }

    private void dfs (TreeNode root, List<Integer> ret) {
        if (root != null) {
            dfs(root.left, ret);
            ret.add(root.val);
            dfs(root.right, ret);
        }
    }

    public List<Integer> inorderTraversal (TreeNode root) {
        List<Integer> ret = new ArrayList<>();
        dfs(root, ret);
        return ret;
    }
}

Iterative:

import java.util.ArrayList;
import java.util.List;
import java.util.Stack;

class BinaryTreeInorderTraversal_Iterative {
    // TreeNode defination
    class TreeNode {
        int val;
        TreeNode left;
        TreeNode right;
        public TreeNode (int _val) {
            val = _val;
        }
    }

    // Iterative
    public List<Integer> inorderTraversal (TreeNode root) {
        List<Integer> ret = new ArrayList<>(); // store result
        Stack<TreeNode> stack = new Stack<>(); // When implementing iteratively, we need stack
        TreeNode cur = root;
        while (!stack.isEmpty() || cur != null) {
            while (cur != null) {
                stack.push(cur);
                cur = cur.left;
            }
            cur = stack.pop();
            ret.add(cur.val);
            cur = cur.right;
        }
        return ret;
    }
}

读代码过程中的思考： 首先，一个节点有两个身份，既是上一个节点的子节点，也是下一个节点的父节点。 其次，上面这个方法使用的是迭代（Iterative），在Iterative中，我们要自己使用栈。所以Iterative方法最难的地方就是想清楚节点的入站出站顺序。

解题过程：

什么是Inorder Traversal？ 本题为中序遍历（Inorder Traversal），即root左侧的节点在遍历后的数组中都要在root的左侧，root右侧的节点都要在root的右侧，这便是Inorder Traversal。这个定义并不难，如果手边有纸其实可以很容易写出一个树的中序遍历结果。

什么是Iterative方法？ 首先要知道与Iterative对应的方法是Recursive。 Recursive是递归，递归和迭代相似性很大但是有区别，这个区别展开来讲可以写一个文章了，所以这里不展开讲，可以自行搜索这个问题，网上有很多解答，并且我也不是那么懂。我只说说我最粗浅的看法：对于中序遍历，我们很容易想到这样一种方法，假设我们有一个函数fun()，这个函数的输入为一个节点，输出为以这个节点为根节点的树的中序遍历结果，那么我们只要不停的对每一个节点和它的子节点都使用这个函数，就像一棵树从上到下不停的往下调用这个函数，那么就可以得到整个树的中序遍历的结果。用代码表示就有点像fun(fun(fun(fun(root))))这种感觉。 Iterative则不同，Iterative并不是完全的像上面那样一层一层调用的那种感觉，虽然iterative里也经常出现不停的调用这种方式，但本质上是有区别但。使用Iterative方法我们需要自己定义栈（stack），将所有即诶但但，并且自己定义节点的出入栈顺序，这个顺序也是最难的一点。 我现在对这个方法对理解程度仅限于：我能解释清楚为什么这个方法可以达到中序遍历对目的，但是我想不到作者是如何想到这个方法的。 但是真想搞明白就要知道作者的思考过程，这里不是说要去问作者它是怎么想的，而是我们自己要研究明白作者的思考过程，或者是一种你自己的思考过程如何引出上面的代码的。
